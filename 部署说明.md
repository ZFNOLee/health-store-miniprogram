## 🚀 快速部署

### Linux / Mac 系统

```bash
# 1. 解压部署包
tar -xzf health-store-miniprogram.tar.gz
cd health-store-miniprogram

# 2. 运行部署脚本
chmod +x deploy.sh
./deploy.sh

# 3. 访问应用
# 浏览器打开：http://localhost:8080/admin-page.html
```

### Windows 系统

```
1. 解压部署包
   右键 → 解压到当前文件夹

2. 双击运行
   deploy.bat

3. 访问应用
   浏览器打开：http://localhost:8080/admin-page.html
```

## 🎮 部署步骤（手动）

如果自动脚本失败，可以手动部署：

### 步骤 1：安装依赖

```bash
cd server
npm install
cd ..
```

### 步骤 2：初始化数据库

```bash
cd server
node scripts/init-db.js
cd ..
```

### 步骤 3：启动后端

```bash
cd server
node index.js &
# 或 Windows: start node index.js
cd ..
```

### 步骤 4：启动 HTTP

```bash
python3 -m http.server 8080 &
# 或 Windows: python -m http.server 8080
```

### 步骤 5：验证

```bash
# 检查后端
curl http://localhost:3000/api/health

# 检查前端
curl http://localhost:8080/test-page.html
```

## 🔧 常见问题

### Q1: 端口被占用

**错误：** `Error: listen EADDRINUSE`

**解决：**
```bash
# 查找占用进程
lsof -i :3000
lsof -i :8080

# 杀死进程
kill -9 <PID>
```

### Q2: npm install 失败

**解决：**
```bash
# 清除缓存
npm cache clean --force

# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# 重新安装
npm install
```

### Q3: 数据库初始化失败

**解决：**
```bash
# 创建数据库目录
mkdir -p database

# 重新初始化
cd server
node scripts/init-db.js
cd ..
```

### Q4: 页面无法访问

**检查：**
```bash
# 检查服务是否运行
ps aux | grep node
ps aux | grep python

# 检查端口
netstat -tlnp | grep 3000
netstat -tlnp | grep 8080

# 查看日志
tail -f /tmp/health-server.log
tail -f /tmp/health-http.log
```
