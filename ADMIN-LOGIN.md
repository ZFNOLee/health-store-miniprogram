## 🚀 登录管理后台

### 方式一：浏览器访问（推荐）

**1. 打开管理后台页面**
```
http://localhost:8080/admin-page.html
```

**2. 选择账号**
- 下拉选择「管理员」或「店员」

**3. 点击登录**
- 测试账号无需密码，直接登录

**4. 登录成功**
- 进入管理后台首页
- 查看统计数据
- 使用各项管理功能

---

### 方式二：API 登录

```bash
# 管理员登录
curl -X POST http://localhost:3000/api/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "openid": "admin_openid_001",
    "nickname": "管理员"
  }'

# 店员登录
curl -X POST http://localhost:3000/api/users/login \
  -H "Content-Type: application/json" \
  -d '{
    "openid": "staff_openid_001",
    "nickname": "店员"
  }'
```

**响应示例**:
```json
{
  "success": true,
  "data": {
    "id": 2,
    "openid": "admin_openid_001",
    "nickname": "管理员",
    "role": "admin",
    "member_level": "diamond",
    "available_points": 50000
  }
}
```

## 🔑 权限说明

### 管理员 (admin)
- ✅ 所有功能权限
- ✅ 商品管理
- ✅ 用户管理
- ✅ 订单管理
- ✅ 积分管理

### 店员 (staff)
- ✅ 商品管理
- ✅ 订单管理
- ⚠️ 部分用户管理（只读）
- ⚠️ 部分积分管理（只读）

### 普通用户 (user)
- ❌ 无法登录管理后台
- ✅ 只能使用用户端功能

## ⚠️ 注意事项

1. **账号安全**
   - 测试环境无需密码
   - 生产环境需要添加密码验证
   - 建议添加 JWT token 认证

2. **权限控制**
   - 当前通过角色字段控制
   - 前端验证 + 后端验证
   - 建议添加更细粒度的权限系统

3. **数据修改**
   - 管理后台的修改会实时同步
   - 用户端数据会立即更新
   - 操作前请确认

## 📞 遇到问题？

### 问题 1: 无法登录
**解决**: 检查后端服务是否运行
```bash
curl http://localhost:3000/api/health
```

### 问题 2: 权限不足
**解决**: 确认账号角色是否为 admin 或 staff
```bash
curl http://localhost:3000/api/users/2
```

### 问题 3: 页面空白
**解决**: 检查 HTTP 服务是否运行
```bash
curl http://localhost:8080/admin-page.html
```
