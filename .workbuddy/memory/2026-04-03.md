## 修复管理员登录权限问题

### 问题根因
- `init-db.js` 初始化时只创建了 `test_openid_123` 普通用户，`admin_openid_001` 没有被预置
- 首次登录时 login 接口自动创建了新账号，默认 role='user'，导致权限校验失败

### 修复内容
1. 运行 `server/scripts/fix-admin-roles.js` 直接修复数据库中角色
2. 修复 `server/routes/users.js` 的 login 逻辑：已存在用户登录时不覆盖 role/member_level，用参数化查询防止编码问题
3. 修复 `server/scripts/init-db.js`：重置数据库时预置 admin/staff/user 三个测试账号，角色正确
4. 重启服务使修改生效

- Python 未安装，使用自建 serve-static.js 替代 `python -m http.server`
- Node.js 版本：v22.22.1，npm 版本：10.9.4
